# 飞书股票智能机器人项目规则

## 1. 项目结构规则

### 1.1 文件组织

- **核心模块**：直接放在项目根目录
- **配置文件**：使用 `config.py` 管理配置，敏感信息通过 `.env` 文件加载
- **依赖管理**：使用 `requirements.txt` 管理所有依赖包
- **测试文件**：以 `test_` 开头的文件用于测试
- **文档文件**：根目录下的 `.md` 文件作为项目文档

### 1.2 目录规范

```
coze_wx/
├── config.py              # 配置管理
├── coze_manager.py        # Coze上下文管理
├── feishu_messenger.py    # 飞书消息推送
├── stock_scheduler.py     # 定时任务调度
├── feishu_websocket.py    # 飞书WebSocket服务器
├── image_analyzer.py      # 图片分析模块
├── main.py                # 主程序入口
├── requirements.txt       # 依赖包
├── .env                   # 环境变量（不提交到版本控制）
├── .env.example           # 环境变量示例
└── README.md             # 说明文档
```

## 2. 代码规范

### 2.1 命名规范

- **文件命名**：使用小写字母和下划线 `_` 分隔
- **类命名**：使用驼峰命名法（如 `CozeManager`）
- **函数命名**：使用小写字母和下划线 `_` 分隔（如 `send_message`）
- **变量命名**：使用小写字母和下划线 `_` 分隔（如 `app_id`）
- **常量命名**：使用全大写字母和下划线 `_` 分隔（如 `DEFAULT_STOCK_QUESTION`）

### 2.2 代码风格

- **缩进**：使用4个空格缩进
- **行宽**：每行不超过100个字符
- **空行**：函数和类之间使用两个空行，函数内部逻辑块之间使用一个空行
- **注释**：关键逻辑添加注释，复杂函数添加文档字符串

### 2.3 导入规范

- 按以下顺序导入模块：
  1. 标准库模块
  2. 第三方库模块
  3. 本地模块
- 使用相对导入时，确保路径正确

## 3. 配置管理规则

### 3.1 环境变量

- **敏感信息**：所有敏感信息（如API密钥、令牌）必须通过 `.env` 文件管理
- **配置分离**：配置分为静态配置（`config.py`）和环境变量（`.env`）
- **默认值**：为所有配置项提供合理的默认值

### 3.2 配置项

- **Coze API配置**：`COZE_API_URL`、`COZE_BEARER_TOKEN`、`COZE_PROJECT_ID`
- **飞书API配置**：`FEISHU_APP_ID`、`FEISHU_APP_SECRET`、`FEISHU_VERIFICATION_TOKEN`、`FEISHU_ENCRYPT_KEY`
- **定时任务配置**：`SCHEDULE_TIMES`
- **业务配置**：`DEFAULT_STOCK_QUESTION`等

## 4. 安全规则

## 6. 部署指南

### 6.1 环境准备

- **Python版本**：3.7+
- **依赖安装**：`pip install -r requirements.txt`
- **环境变量**：复制 `.env.example` 为 `.env` 并配置

### 6.2 运行模式

- **前台运行**：`python main.py`
- **后台运行**：使用 `pythonw` 或服务管理工具
- **Windows服务**：使用 NSSM 注册为Windows服务

### 6.3 监控和维护

- **日志监控**：定期查看日志文件
- **服务状态**：确保服务正常运行
- **依赖更新**：定期更新依赖包

## 7. 故障排查流程

### 7.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无法连接飞书WebSocket | 网络问题或配置错误 | 检查网络连接和配置 |
| Coze API调用失败 | Token无效或网络问题 | 检查Token和网络连接 |
| 定时任务不执行 | 系统时间错误或配置问题 | 检查系统时间和配置 |
| 消息推送失败 | 权限问题或网络问题 | 检查权限和网络连接 |
| 图片分析失败 | API调用失败或图片格式问题 | 检查API配置和图片格式 |

### 7.2 排查步骤

1. **查看日志**：检查 `feishu_bot.log` 文件中的错误信息
2. **验证配置**：确认所有配置项正确设置
3. **测试API**：验证API调用是否正常
4. **检查网络**：确认网络连接正常
5. **重启服务**：尝试重启服务解决问题
